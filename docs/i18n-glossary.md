# Arabic i18n glossary & conventions (TASK-1604)

The reference for every Arabic string in `src/i18n/messages/ar.json`. When adding a
new key, translate with these terms and conventions so the catalog stays consistent.
Register: Modern Standard Arabic in the mainstream sports-media style (beIN / FilGoal
register) — formal but not stiff.

## Core conventions

| Convention           | Decision                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Brand**            | `PitchIQ` stays **Latin** everywhere (matches the logo wordmark; the earlier transliteration بيتش آي كيو was dropped). `VAR` also stays Latin when parenthesised: الفار (VAR).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Competition name** | **الدوري الإنجليزي الممتاز** — used in headings, meta titles/descriptions, and body copy. The bare الدوري الإنجليزي is allowed only where space is genuinely tight.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Numerals**         | **Eastern-Arabic numerals (١٢٣)** everywhere (owner decision, Session 59 — supersedes the earlier Western-digit choice). Applied via `numberingSystem: "arab"` in the Intl formatters + `localizeDigits()` for raw JSX numbers **and for bare ICU `{arg}` numbers** (a bare `{arg}` is String()'d, not number-formatted, so pass `localizeDigits(n, locale)`). **⚠️ Known gap (follow-up, bundle with trivia):** ICU **plural `#`** counts + a couple hardcoded numbers inside messages still render **Latin** on `/ar` — the `#` uses the locale's default NumberFormat (Latin for bare `ar`), and we can NOT use the `ar-u-nu-arab` formatting-locale trick because next-intl's locale-aware `usePathname`/`Link` break on a non-routing locale (the locale switcher stops working). Fix later by rewriting each `#` plural to a `{n}` display arg + passing `localizeDigits(count)` alongside `count`. Affected: `playersCount`, `statSubline`, `titlesBadge`, `clubCountAria`, map `clubsActive`/`regionSummary`/`titleBadge`. **Bidi:** season labels + dates are **NOT** bidi-isolated on `/ar` — they flow naturally RTL so an Arabic reader reads them correctly (the visual "٢٠٢٦ - ٢٠٢٥" reads right-to-left as 2025→2026; a DOB reads day-on-right). Do NOT force LTR / add isolates around these. **Season label:** on `/ar` it's a FULL, spaced range — `٢٠٢٥ - ٢٠٢٦` (both years, spaces around the dash), via `formatSeasonLabel`. **Grouping separator:** `formatNumber` groups with a **Latin baseline comma** on `/ar` (e.g. capacity `٣١,٧٥٠`), NOT the CLDR Arabic high separator U+066C `٬` (owner: it reads as floating above the number). |
| **Quotes**           | Arabic guillemets **«…»** around user queries and quoted terms (`noMatchesFor`, `noComparisonMsg`, …).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Counted nouns**    | Any string interpolating a count uses a full **ICU plural** with the Arabic categories (`zero/one/two/few/many/other`): `one` → واحد/واحدة (no `#`), `two` → dual (no `#`), `few` (3–10) → `# ` + plural, `many` (11–99) → `# ` + singular accusative (…ًا), `other` → `# ` + singular. Never place a **definite** noun directly after a number.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Label + number**   | Where a label is composed next to a raw number by code (`leaderboard.value`), the Arabic string flips to **"{label}: {value}"** (e.g. الأهداف: 25) so no grammatical-number agreement is needed.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **"Home" concept**   | The dashboard/home page is **الرئيسية** everywhere (nav, back-links, the dashboard meta title).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

## Data stays in source form

Team, player, and manager **names**, free-string positions on profiles, and the
"2025-26" season labels render untranslated by design (TASK-1606 is the owner-gated
transliteration project; season labels/dates/numbers are TASK-1605).

## Formatting (resolved — TASK-1605)

All of the below now route through the locale-aware `src/utils` formatters
(`format.ts` `isRtl`/`bidiIsolate`/`formatNumber`, plus `formatSeasonLabel` /
`formatKickoff` / `formatShortDate` / `formatMatchdayHeading` /
`relativeTimeFromNow` / `formatBirthDate`, each with a `locale` param defaulting
to English). Numbers + season labels keep an identical Western-digit value in
both locales and are **bidi-isolated** (`U+2066…U+2069`) in RTL so an LTR run
(e.g. "2025-26") can't visually reverse; dates + relative time genuinely localize
(Arabic weekday/month names + "أمس"/"منذ 3 أسابيع", Latin digits).

- `footer.dataUpdated` — Arabic relative time via `Intl.RelativeTimeFormat`.
- Trivia provenance season label — bidi-isolated; the range is isolated as one unit.
- Season labels ("2025-26"), `ordinal(rank)` (→ plain Western number on `/ar`),
  attendance/capacity/stat-tile numbers, kickoff/matchday dates, and DOB/DOD dates.

**⚠️ RTL numeric dates (Session 60 cont.):** a slash-separated numeric date
(`DD/MM/YYYY`) resolves to a **single left-to-right number-island** under the
Unicode bidi algorithm, so it always displays in logical order regardless of
paragraph direction. To make it read **day → month → year right-to-left** (day
adjacent to the ولد/تُوفّي label), `formatBirthDate` emits **`YYYY/MM/DD`** order
on RTL (the island then shows year-on-the-left, day-on-the-right). This is the
whole fix — do NOT add bidi controls. Season **ranges** (`٢٠١٩–٢٠٢٦`, `٢٠٢٢–٢٠٢٥`)
similarly just flow RTL (compact year range, no isolate).

**⚠️ Tailwind v4 `rtl:-scale-x-100`** (mirror a directional icon like the map
play triangle) lands on the separate **`scale`** CSS property, NOT `transform` —
verify with `getComputedStyle(el).scale === "-1 1"`.

**Out of scope (stay English by design):** the OG-image routes (build-time PNGs,
no locale context).

## Trivia facts (localized — Session 60)

All 26 "Did you know?" rules emit an optional `key` (a `trivia.fact*` message key)

- `values`; `<TriviaCard>` renders `t(key, values)` on **RTL only** (English still
  uses the rule's source-form `text`), so `/en` is untouched. `localizeFactValues`
  locale-formats the params: numbers → Eastern-Arabic, a **`season`** value →
  `formatSeasonLabel`, numbers **≥ 1000** (attendances) → grouped baseline comma;
  entity names (team/player/manager) pass through source-form. **Trivia messages use
  NO ICU plural selectors** — the card pre-localizes every number value to a STRING,
  so a `{arg, plural}` selector would break AND a `#` would render Latin; handle
  Arabic number-noun agreement by phrasing (accusative singular for the common
  11-99 range). Scorelines inside a fact are bidi-isolated in the message
  (`⁦…⁩`). Ordinals (goal-extremes "second-fewest", position-record rank)
  read a simplified Arabic form; English keeps the precise ordinal via `text`.

## ICU `#` / bare-count args (localized — Session 60)

next-intl formats an ICU plural `#` and a bare `{count}` with the locale's default
NumberFormat = **Latin on bare `ar`** (the `ar-u-nu-arab` config-locale trick is
banned — it breaks the locale-aware `Link`/`usePathname` nav). Fix: the **Arabic**
plural displays a **pre-localized string arg** in place of `#`, keeping the numeric
arg only as the plural SELECTOR; the consumer passes `localizeDigits(count, locale)`
alongside `count`. English messages keep `#`/`{count}` (Latin is correct there).
